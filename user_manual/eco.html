ECO

Engineering change orders (ECO) is process in Medtrack system which allows users in the system to release a part/document created by them through approval mechanism.



Create ECO

1.a The user navigates to 'ECO' on side navigation clicks on 'New' button to create a new ECO.

1.b User enters all mandatory fields marked as * , clicks on 'Save' button . A new ECO created successfully with a unique ECO number and ECO status is pending.

1.c To add 'Part/Document' and assign 'Approvers' to the ECO user clicks on ‘Edit’ button on ECO details page.

1.d To add ‘part/Document’ click tab 'Part/Document under ECO' select part/ document for ECO by clicking dropdown icon and clicks on ‘+’ for adding the part/document to ECO, click on ‘remove’ icon for removing a part/ document from ECO.You can add multiple ‘part/document’ by following the above process.

1.e To add approvers click tab ‘Approvers’, select 'Approver’s Department', 'Approvers name' and clicks “+” for adding approver to ECO, clicks on ‘remove’ icon for removing an approver from ECO, click on ‘Save’ button to save the ECO.The ECO saved with a part/ document and approver list and ECO status is ‘Pending’.

View ECO

2.a Navigate to ‘ECO’ from side navigation , click on text under ‘ECO title’ column in list view table.
2.b Click on ‘Description’ tab to view description of the ECO.
2.c Click on ‘Part/Document’ tab to view parts/documents under ECO'.
2.c Click on ‘Part/Document’ tab to view parts/documents under ECO'.
Edit ECO
3.a User clicks 'Edit' button, the system opens the ECO in editable mode.
3.b User can add ‘Long Description’ in ‘Description’ tab.

3.c User can navigate to other tabs in editable mode, by navigating to ‘Part/Documents under ECO’ user can add and remove parts/Documents.
3.d To edit approvers navigate to Approvers tab, add or remove approvers in the tab.

3.e Click on ‘Save’ button to save details.
3.f Click on ‘Submit’ button to submit the ECO after editing an ECO. The status of the ECO is changed to ‘Submitted’.
List of ECO.

4.a Navigate to ‘ECO’ from side navigation.
4.b The system displays list of all ECO’s,
4.c Type search text in search box highlighted and the system displays a filtered list based on the search text entered.

4.d The user can sort ECO list based on ‘ECO#’, 'ECO title', 'status', 'created by','created on' by clicking on sort icon which appears when you hover on column title of the table.
4.e ECO list can be filtered in a date range of created date.
4.f Clicking on the status checkboxes as highlighted in the image, user can filter list values based on ECO status.
4.g User can quickly get a list of all ECO’s waiting for his attention by clicking “waiting for me” toggle on.
Approve/ Reject/ Void By DCA
5.a DCA clicks on the approve button to approve the ECO.
5.b Types comment, password (digital signature), clicks ‘approve’, notification send to the creator and approvers of this ECO.
5.c The ECO status changed to 'under_approval' from 'submitted'.
5.d DCA clicks on the reject button to reject the ECO.
5.e Types comment, password (digital signature), clicks ‘reject’,the Eco status changed to ‘pending’.
5.f DCA can also void the ECO by clicks on the void button.
5.g Types comment, password (digital signature), clicks ‘void’,the Eco status changed to ‘void’.
Approve/ Reject by Approver

6.a Click on ECO title of the ECO’s which are currently in ‘under-approval’ status from ECO list . Details page of that ECO displays three action buttons 'Approve', 'Reject', and 'Back' button if the logged in user is approver of the ECO.
6.b User can click on the 'Approve' button for approve the ECO.
6.c System displays a pop up prompting user to enter ‘comment’ and ‘password’ to approve ECO.
6.d 'Approver action' status changes to 'Approved' from 'Pending after approve action from approver.
6.e The ECO status is changed to 'approved' from 'under_approval' after all the approvers of the ECO have approved the ECO
6.f Approvers can also reject the ECO by click on the reject button, types comment and password, click reject button.
6.h The ECO status is changed to ‘pending’ form ‘under_approval’.
ECO Released by DCA.

7.a Click on ECO title of the ECO’s which are currently in ‘approved’ status from ECO list.Click on Approvers tab in ECO details page if the user type is DCA ‘Release’ button is enabled.
7.b User types Released Date and password (digital signature), clicks on the ‘Release’ button.
7.c ECO released successfully and the ECO status changes to ‘Complete’ from ‘Approved’.
