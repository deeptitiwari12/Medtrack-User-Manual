Users
The ‘User’ module allows to creates a new, edit, view user, and a list, the module is available only to DCA (Document Control Admin).


Create User
1.a To create a new user, admin navigates on 'User' -> 'New' from the user list.
1.b Admin enters data on mandatory fields.

1.c The mandatory fields are first name, last name, designation, email, password, mobile, hire date, and department(s).

1.d Admin allows to create a user as admin by set Is Admin = Yes.

1.e Admin can add a new department by clicks on '++', type department name, press enter key.
1.f Admin allocates department 'Quality Assurance', 'Operations', and 'Finance' to David Villa
1.g Clicks on remove, 'Finance' wipes out from department list.
1.h Admin clicks on 'Save'. The user 'David Villa' saved, and display on the user list.

View User

2. View user details with ‘Notify User’, ‘Edit’, and ‘Back’.

2.a Admin clicks on username 'David Villa' on the list, the system moves to user details view page ('David Villa').
2.b Admin clicks on ‘Notify User’, the system sent an email notification to 'David Villa'.

2.c ‘Back’ moves to the user list.

Edit User

3. Edit user details with 'Save', 'Reset Password', 'Cancel'.

3.a Admin clicks on 'Edit', the system opens to edit user details of 'David Villa'.


3.b The system displayed automated password when admin clicks on 'Reset password'.
3.c Admin changed ‘Mobile No’ to ‘9874563212’ from ‘9874563210’, and clicked on ‘Save’, the user, David's mobile number

updated successfully.

List of Users

4. Admin views ‘User list’

4.a Admin can set status 'active' to 'inactive' or vice versa. An inactive user status indicates that the user is not acting with

the organization, the default is active.
4.b Admin types 'John', clicks on search, the system displayed searched list.
4.c Admin clicks on sort button of name, the system displayed sorted list. The system also allow sort on 'Email', 'Status',

'Is active'.
